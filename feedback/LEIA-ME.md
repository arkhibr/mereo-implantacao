# feedback/

Comunicação **recebida do cliente** sobre o material entregue — o lado *inbound*
que fecha o ciclo de cada pasta de `entregas/`.

## Organização

```
feedback/<cliente>/<assunto>_<AAAA-MM-DD>.pdf
```

- `<cliente>` — nome do cliente (ex.: `belmont`).
- `<assunto>` — módulo/tema a que o feedback se refere (ex.: `competencias`).
- `<AAAA-MM-DD>` — data em que o feedback foi **recebido**.

## Convenção

- Aqui entra o que o cliente **devolve** (feedback, validações, pedidos de
  ajuste). O que **nós enviamos** mora em `entregas/`.
- Arquivos são insumo de entrada — não editar; se houver versão revisada,
  adicionar um novo arquivo com nova data, não sobrescrever.
